# 通用电商 9 图系统 V2.0

> Universal Ecommerce Image System

## Purpose

建立一套可跨平台、跨品类复用的电商 9 图体系。九张图片不只是展示产品，而是沿着消费者的购买决策路径，依次完成吸引、理解、选择、欲望、品质证明、使用证明、身份认同、价值提升与信任建立。

## Scope

适用平台：

- Shopee
- TikTok Shop
- Lazada
- Amazon（需按平台规则调整）
- 独立站

适用品类包括甲油胶、美甲工具、睫毛产品、护肤品、宠物用品、家居用品、小家电及其他标准化电商产品。

本文定义 9 图的固定顺序、营销职责、通用输入、视觉一致性和验收边界。具体品牌风格由 [`brand_style_guide_品牌视觉规范.md`](./brand_style_guide_品牌视觉规范.md) 约束；具体 Prompt 内容不在本文定义。

## 1. 核心原则

电商图片应服务于购买决策，而不是简单堆叠产品信息。

```text
停止滑动
→ 了解产品
→ 选择 SKU
→ 产生购买欲
→ 相信品质与使用体验
→ 认同品牌与价值
→ 消除顾虑并下单
```

执行时遵循以下原则：

1. 九张图各自承担一个明确的营销任务。
2. 图片顺序和营销目标固定，产品素材、文案与 SKU 信息可变。
3. 每张图只保留完成当前任务所需的信息。
4. 所有图片共享同一品牌色彩、字体、Logo、光影和构图逻辑。
5. 内容必须适合移动端快速浏览，关键信息应在缩略图和 3 秒浏览中可识别。

## 3. 单图规范

### 01 Stop Scroll｜主图

- **目标：** 让用户停下来并产生点击意愿。
- **用户问题：** 这是什么？为什么值得我看？
- **必要内容：** 产品主体、Logo、产品名称、一个核心卖点。
- **构图要求：** 产品占画面约 60%–80%，视觉冲击最强，缩略图下仍可识别。
- **验收重点：** 产品是唯一视觉中心；标题和核心卖点不抢夺主体。

### 02 Explain｜卖点图

- **目标：** 在 3 秒内解释购买理由。
- **用户问题：** 它能为我解决什么问题？
- **必要内容：** 3–5 个经过确认的核心卖点，可搭配统一风格图标。
- **构图要求：** 信息层级清楚，短句优先，避免大段文字。
- **验收重点：** 卖点真实、易读、无重复，且与产品素材和商品信息一致。

### 03 Choose｜SKU 图

- **目标：** 降低选择成本，帮助用户确认合适的 SKU。
- **用户问题：** 我应该选哪个颜色、型号、尺寸或套餐？
- **必要内容：** 色号、规格、尺寸、型号或套餐关系。
- **品类适配：** 甲油胶展示色卡；工具展示型号；其他品类展示可比较的 SKU 属性。
- **验收重点：** SKU 名称、颜色与顺序准确，不得生成不存在的变体。

### 04 Desire｜效果图

- **目标：** 展示最终效果，激发拥有欲。
- **用户问题：** 使用后会得到什么结果？
- **必要内容：** 产品带来的可视化结果。
- **品类适配：** 甲油胶展示上手效果；工具展示处理结果；宠物用品可展示真实且可证明的使用前后差异。
- **验收重点：** 效果可信，不夸大、不伪造产品能力。

### 05 Quality｜细节图

- **目标：** 用可见细节证明品质。
- **用户问题：** 它的材质、工艺和结构可靠吗？
- **必要内容：** 材质、工艺、纹理、刷头、瓶口或产品结构中的关键细节。
- **构图要求：** 使用微距或局部特写，保留真实材质反馈。
- **验收重点：** 细节清晰，结构未被 AI 改写，反射与纹理符合实物。

### 06 Use｜使用图

- **目标：** 证明产品在真实场景中易用、顺手。
- **用户问题：** 它怎么使用？使用过程是否自然？
- **必要内容：** 正确的操作方式、人物或使用环境、产品与动作的关系。
- **品类适配：** 美甲师操作甲油胶、使用修甲工具、为宠物梳毛等。
- **验收重点：** 动作合理，产品尺寸、握持和使用方式符合现实。

### 07 Aspire｜向往图

- **目标：** 塑造理想身份、环境和生活方式。
- **用户问题：** 拥有它之后，我会成为怎样的人？
- **必要内容：** 与品类相关的高级环境，如美甲工作室、宠物美容店或家居场景。
- **构图要求：** 环境强化产品价值，但不能淹没产品主体。
- **验收重点：** 场景与目标用户相关，品牌气质一致，不使用无关奢华元素。

### 08 Premium｜价值图

- **目标：** 通过完整体验提升感知价值。
- **用户问题：** 它是否适合收藏、送礼或作为高价值套装购买？
- **必要内容：** 真实存在的礼盒、包装、收纳、开箱或套装组合。
- **构图要求：** 强调秩序、完整度与开箱体验。
- **验收重点：** 不虚构包装和套装内容，画面中的物品必须与实际交付一致。

### 09 Trust｜信任图

- **目标：** 消除购买前的最后顾虑。
- **用户问题：** 品质、服务与承诺是否可信？
- **可用内容：** 已验证的品质检测、品牌保障、售后服务、专业背书或认证信息。
- **构图要求：** 信息明确、克制，避免制造虚假权威感。
- **验收重点：** 只有具备证据的认证、检测与承诺才可出现；禁止 AI 编造徽章、证书或功效声明。

## 4. 视觉一致性

九张图必须共享：

- 同一色彩体系
- 同一字体体系
- 同一 Logo 规则
- 同一光影语言
- 同一构图逻辑
- 同一素材真实性标准

禁止每张图采用完全不同的风格、混用不相关字体或使用冲突的光影体系。品牌级细则以品牌视觉规范为准。

## 5. Prompt 组成模型

每张图的 Prompt 由四部分组合：

```text
固定视觉规范
+ 当前阶段任务
+ 产品素材与事实
+ 产品卖点及 SKU 数据
```

- **固定视觉规范：** 长期稳定的品牌视觉语言和负面约束。
- **当前阶段任务：** 九图中当前图片的唯一营销目标。
- **产品素材与事实：** 产品、Logo、包装、结构和场景参考。
- **卖点及 SKU 数据：** 已审核的文案、颜色、尺寸、型号和套餐信息。

具体目录和文件约定见 [`../prompts/README.md`](../prompts/README.md)。

## 6. 输入、输出与固定边界

### 必要输入

- 产品图片
- Logo 文件
- 产品名称
- 已审核卖点
- SKU 数据
- 品牌或联名品牌标识

### 预期输出

- 按 01–09 固定顺序组织的一套 9 图资产
- 每张图对应唯一营销目标
- 所有图片通过阶段验收与品牌一致性检查

### 允许变化

- 产品素材
- 经审核的文案
- SKU 信息
- 与品类匹配的使用和向往场景

### 禁止变化

- 九图顺序
- 每张图的营销目标
- 产品外观、Logo、标签、包装和 SKU 颜色等事实
- 已批准的品牌视觉规范

## 7. 完成标准

一套 9 图只有在以下条件全部满足时才可进入后续流程：

- 九张图齐全且顺序正确。
- 每张图只承担其指定营销任务。
- 产品、Logo、包装、SKU 和声明均与输入事实一致。
- 视觉风格统一并符合品牌规范。
- 文字适合移动端阅读，无明显信息拥挤。
- 不含虚假认证、夸大效果、错误结构或额外产品。

## 8. 后续方向

本规范为生成、审核、上传与发布流程提供统一基础。自动化实现、平台适配和具体 Prompt 库应在后续版本中分别设计，不在 V2.0 文档范围内。
