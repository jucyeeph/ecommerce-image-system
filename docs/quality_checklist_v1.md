# Quality Checklist V1

## Purpose

为单张图片和整套 9 图提供统一人工审核清单。评分定义以 [`../specs/quality_check_rules.yaml`](../specs/quality_check_rules.yaml) 为机器可读来源。

## Scoring

- 每项按 0–5 分评分。
- 单项 4 分及以上为通过。
- 加权平均至少为 4.0。
- 标记为关键项的规则必须全部通过，不能用其他高分抵消。

## Checklist

### Logo｜关键项

- [ ] 使用官方 Logo 源文件。
- [ ] 比例、颜色、字距和安全区正确。
- [ ] Logo 清晰且未被裁切或重绘。

### 产品真实性｜关键项

- [ ] 产品形状、标签和包装与素材一致。
- [ ] SKU 颜色、数量和配件准确。
- [ ] 不存在 AI 生成的额外结构或产品。

### 品牌一致性

- [ ] 色彩、字体、材质和视觉关键词符合品牌规范。
- [ ] 九张图属于同一视觉体系。

### 构图

- [ ] 只有一个视觉中心。
- [ ] 产品比例符合当前图片任务。
- [ ] 关键信息位于平台安全区。
- [ ] 没有无关道具和信息堆叠。

### 灯光

- [ ] 光源方向一致。
- [ ] 材质、轮廓和细节清楚。
- [ ] 高光未过曝，反射符合物理逻辑。

### 文字｜关键项

- [ ] 拼写、语法和目标语言正确。
- [ ] 卖点、SKU、认证和服务信息可核验。
- [ ] 无水印、乱码或伪造文字。

### 移动端可读性

- [ ] 缩略图下产品仍可识别。
- [ ] 主信息无需放大即可阅读。
- [ ] 信息密度适合快速扫读。

### 营销目标达成度｜关键项

- [ ] 图片符合对应 01–09 阶段。
- [ ] 只承担一个主要营销目标。
- [ ] 对应阶段的必要输出已出现。

## Decision

- **Pass：** 所有关键项通过，其他项达到分数线。
- **Revise：** 无关键错误，但一个或多个普通项低于 4 分。
- **Reject：** 任一关键项失败、缺少证据或出现未知 SKU / 声明。
