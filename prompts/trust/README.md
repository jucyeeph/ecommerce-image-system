# 09 Trust｜信任图 Prompt 规范

## Objective

使用可核验的信息消除用户对品质、安全、服务和购买保障的顾虑。

## Required Inputs

- 已验证的认证或检测资料
- 品质保障说明
- 售后政策
- 可使用的品牌或专业背书

## Visual Requirements

- 信息清楚、克制、具有正式感。
- 认证、检测、保障和服务各自来源明确。
- 图标与文字遵循品牌视觉规范。
- 重点信息适合移动端阅读。

## Forbidden

- AI 编造证书、徽章、检测结果或专业背书
- 未经批准的医疗、安全或功效声明
- 模糊来源的绝对化承诺
- 用伪官方视觉制造权威感

## Acceptance Checklist

- [ ] 每项声明均有可核验依据。
- [ ] 认证名称、标志和适用范围准确。
- [ ] 售后与保障内容符合实际政策。
- [ ] 画面没有虚构权威或夸大承诺。
